# Seedance 2.0 取消任务 API 整理

来源：火山引擎方舟官方文档「取消或删除视频生成任务」

原文链接：https://www.volcengine.com/docs/82379/1521720?lang=zh

官方更新时间：2026-04-25 18:45:06（北京时间）

整理时间：2026-05-17

## 1. 接口概览

### 1.1 接口地址

```http
DELETE https://ark.cn-beijing.volces.com/api/v3/contents/generations/tasks/{id}
```

### 1.2 接口作用

这个接口用于两类操作：

1. 取消排队中的视频生成任务
2. 删除已结束任务（成功/失败/超时）的任务记录

注意：这个接口对不同任务状态的行为不同，不是统一“取消”。

### 1.3 鉴权方式

本接口支持 API Key 鉴权。

请求头示例：

```http
Authorization: Bearer ${ARK_API_KEY}
```

## 2. 请求参数

### 2.1 `id`

类型：`string`

必填：是

位置：URL 路径参数（文档也提示属于 URL 查询场景参数）

说明：要取消或删除的任务 ID。

示例：

```http
DELETE https://ark.cn-beijing.volces.com/api/v3/contents/generations/tasks/cgt-20250331175019-68d9t
Authorization: Bearer ${ARK_API_KEY}
```

## 3. 不同状态下的行为矩阵

官方文档给出的关键规则如下：

| 当前任务状态 | 是否支持 `DELETE` | 实际执行动作 | 操作后状态 |
|---|---:|---|---|
| `queued` | 是 | 取消排队中的任务 | `cancelled` |
| `running` | 否 | 不执行取消 | 无变化 |
| `succeeded` | 是 | 删除任务记录（之后不可再查询） | 记录删除 |
| `failed` | 是 | 删除任务记录（之后不可再查询） | 记录删除 |
| `cancelled` | 否 | 不支持再次删除/取消 | 无变化 |
| `expired` | 是 | 删除任务记录（之后不可再查询） | 记录删除 |

## 4. 响应参数

官方文档说明：该接口无返回参数。

工程上通常会看到：

| 情况 | 处理建议 |
|---|---|
| 请求成功 | 按 HTTP 状态码判断成功（一般是 2xx） |
| 请求失败 | 展示错误提示并保留任务状态 |

因为没有业务响应体，客户端需要靠 HTTP 状态码和后续查询接口来确认结果。

## 5. 典型使用场景

### 5.1 用户点击“取消生成”

推荐流程：

1. 先调用查询任务接口拿最新状态
2. 如果状态是 `queued`，允许调用 `DELETE`
3. 调用成功后，把前端状态更新为 `cancelled`，并可短轮询确认
4. 如果状态已是 `running`，提示“任务已开始执行，当前不支持取消”

### 5.2 用户清理历史任务记录

推荐流程：

1. 仅对 `succeeded` / `failed` / `expired` 显示“删除记录”按钮
2. 调用 `DELETE` 成功后从本地列表移除该记录
3. 如果业务侧已有本地文件（视频已下载），仅删除云端任务记录，不自动删除本地文件，避免误删用户素材

### 5.3 避免误操作

建议在 UI 中对不同状态展示不同文案：

| 状态 | 推荐按钮文案 |
|---|---|
| `queued` | 取消任务 |
| `running` | 运行中（不可取消） |
| `succeeded` / `failed` / `expired` | 删除任务记录 |
| `cancelled` | 已取消（不可重复操作） |

## 6. 和查询接口联动建议

由于删除后任务可能无法再查，建议在删除前先保存必要元数据。

删除前建议落盘字段：

```json
{
  "taskId": "cgt-20250331175019-68d9t",
  "model": "doubao-seedance-2-0-260128",
  "status": "succeeded",
  "videoUrl": "https://...",
  "createdAt": 1743414619,
  "updatedAt": 1743414655,
  "deletedAt": 1743414999
}
```

推荐调用顺序：

1. `GET /tasks/{id}`：拿当前状态
2. 根据状态决定是否允许 `DELETE`
3. `DELETE /tasks/{id}`：执行取消或删除
4. 前端列表同步刷新（必要时再次 `GET` 验证）

## 7. 可能的易错点

### 7.1 把 `DELETE` 当成“强制停止运行中任务”

不成立。`running` 状态不支持 `DELETE` 取消。

### 7.2 删除成功任务后还想查详情

不成立。`succeeded`、`failed`、`expired` 删除后，后续将不支持查询该任务记录。

### 7.3 重复删除/重复取消

`cancelled` 状态不支持再次 `DELETE`。前端应禁用重复按钮，避免用户误以为操作失败。

## 8. Cinecho 项目接入建议

如果在 Cinecho 中接入这个 API，建议最少做这些能力：

1. 在任务卡片中根据状态动态展示按钮（取消 / 删除 / 禁用）。
2. 点击按钮前先查一次任务状态，防止状态过期导致操作无效。
3. 对 `running` 状态给出明确提示：“当前不支持取消运行中任务”。
4. 删除记录时只删云端任务记录，不默认删除本地已下载视频文件。
5. 操作后刷新任务列表并落地操作日志，便于排查。

## 9. 最简调用示例

```bash
curl -X DELETE "https://ark.cn-beijing.volces.com/api/v3/contents/generations/tasks/cgt-20250331175019-68d9t" \
  -H "Authorization: Bearer $ARK_API_KEY"
```

## 10. 三个 API 的关系（创建 / 查询 / 取消）

为了工程上闭环，通常按下面模式使用：

1. 创建任务 API：拿到任务 ID
2. 查询任务 API：轮询状态直到终态，拿视频结果
3. 取消任务 API：对排队中任务取消，或对已结束任务清理记录

这样就形成完整任务生命周期管理。
