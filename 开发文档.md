# Cinecho 1.0 极简本地版技术架构与开发详细设计 (Detailed Blueprint)

## 1. 工程目录结构设计 (Project Structure)
采用标准的 Electron + Vite + Vue 3 结构，严格区分主进程、渲染进程与预加载脚本。

```text
cinecho-local/
├── build/                      # electron-builder 打包配置与图标
├── resources/                  # 静态资源
│   └── bin/
│       └── ffmpeg.exe          # Windows 平台打包自带的静态 FFmpeg
├── src/
│   ├── main/                   # 【Node.js 主进程】
│   │   ├── index.ts            # 主程序入口，窗口生命周期管理
│   │   ├── ipcHandler.ts       # 集中注册所有 ipcMain.handle/on 监听
│   │   ├── volcEngine.ts       # 火山引擎 API 鉴权与请求封装
│   │   ├── ffmpegRunner.ts     # fluent-ffmpeg 子进程调用与进度解析
│   │   └── dataManager.ts      # 负责读取与原子化写入 storage.json
│   ├── preload/                # 【预加载脚本】
│   │   └── index.ts            # contextBridge 暴露 window.electronAPI
│   └── renderer/               # 【Vue 3 渲染进程】
│       ├── src/
│       │   ├── assets/         # UI 样式与本地图片
│       │   ├── components/     # shadcn/ui 基础组件库
│       │   ├── stores/         # Pinia 状态管理 (同步主进程数据)
│       │   ├── views/          # 页面：Home, Settings, History
│       │   └── App.vue
│       ├── index.html
│       └── vite.config.ts
├── package.json
└── tsconfig.json
```

## 2. 数据持久化层设计 (JSON Store)
在主进程的 `dataManager.ts` 中管理，这里给出 `storage.json` 的严格 TypeScript 接口定义。

### 2.1 数据结构 (TypeScript Interfaces)
```typescript
// 顶层数据结构
interface CinechoStore {
  settings: UserSettings;
  tasks: AITask[];
  editHistory: EditRecord[];
}

interface UserSettings {
  volcAccessKey: string;    // 敏感，建议主进程使用 safeStorage 加密后存入 JSON
  volcSecretKey: string;    // 敏感，同上
  outputDir: string;        // 默认输出路径，如 "D:\Cinecho\Outputs"
  theme: 'light' | 'dark' | 'system';
}

interface AITask {
  id: string;               // 客户端生成的 UUID
  volcTaskId: string;       // 火山引擎返回的远程任务 ID
  prompt: string;
  sourceImage: string;      // 本地参考图绝对路径
  status: 'PENDING' | 'PROCESSING' | 'COMPLETED' | 'FAILED';
  resultVideoPath: string | null; // 最终下载到本地的绝对路径
  errorLog: string | null;
  createdAt: number;        // 时间戳
}
```

### 2.2 原子化写入策略 (避免断电数据丢失)
为了防止在写入 `storage.json` 时应用崩溃导致文件内容变为空白，主进程必须采用以下安全写入机制：
1. 将内存对象 `JSON.stringify` 写入 `storage.json.tmp`。
2. 写入成功后，使用 `fs.renameSync` 将 `.tmp` 文件重命名为 `storage.json`（在 Windows 系统层面这是一个原子操作）。

## 3. IPC 通信协议定义 (通道注册表)
这是渲染进程 (Vue) 与主进程 (Node) 沟通的唯一桥梁。所有通道通过 `preload.ts` 暴露。

| 通道名称 (Channel) | 发送方 -> 接收方 | 载荷参数 (Payload) | 返回值 (Returns) | 核心职责描述 |
| :--- | :--- | :--- | :--- | :--- |
| `store:get` | Vue -> Node | 无 | `CinechoStore` | 初始化时，Vue 获取全量 JSON 数据装载进 Pinia。 |
| `store:updateSetting` | Vue -> Node | `Partial<UserSettings>` | `boolean` | 保存设置，Node 接收后更新内存并触发 JSON 落盘。 |
| `dialog:selectDir` | Vue -> Node | 无 | `string` (路径) | 调用原生 `dialog.showOpenDialog` 选择视频输出文件夹。 |
| `ai:createTask` | Vue -> Node | `{ prompt, imagePath }` | `string` (UUID) | 提交生成任务，主进程立即生成本地 UUID 返回并开始后台请求。 |
| `video:clip` | Vue -> Node | `{ source, start, end }` | `string` (任务ID) | 提交剪辑任务。 |
| **`sync:taskProgress`** | Node -> Vue | `{ id, status, progress }` | (单向推送) | **双向通信核心**：主进程通过 `webContents.send` 实时向 Vue 推送 API 轮询状态或 FFmpeg 进度。 |

## 4. 核心业务模块底层实现

### 4.1 火山引擎 API 直连封装 (`volcEngine.ts`)
由于没有后端网关代理，客户端需要自己完成复杂的 API 签名（Signature V4）。
1. **读取密钥**：从 JSON 配置中解密提取 `AccessKey` 和 `SecretKey`。
2. **构建请求**：
   * 将用户选择的本地图片读取为 Base64 或二进制流。
   * 构造符合火山豆包视频大模型规范的 JSON Payload。
   * 使用 `crypto` 模块（Node.js 原生）根据火山文档计算 HMAC-SHA256 签名，注入 Headers。
3. **退避轮询机制 (Exponential Backoff)**：
   * 任务提交成功获取 `taskId` 后，开启定时器 `setInterval`。
   * 频率控制：第 1-10 次（每 3 秒），第 11-30 次（每 5 秒），之后（每 10 秒）。
   * 请求状态接口，若返回 `SUCCEEDED`，提取视频 URL。

### 4.2 视频直落本地磁盘下载流
获取到云端视频 URL 后，**切忌**将视频加载到内存中再保存，必须使用流式写入：
```javascript
// 主进程代码示例
const axios = require('axios');
const fs = require('fs');

async function downloadVideo(url, outputPath) {
  const response = await axios({ url, method: 'GET', responseType: 'stream' });
  const writer = fs.createWriteStream(outputPath);
  response.data.pipe(writer);
  return new Promise((resolve, reject) => {
    writer.on('finish', resolve);
    writer.on('error', reject);
  });
}
```

### 4.3 FFmpeg 本地剪辑引擎 (`ffmpegRunner.ts`)
主进程解析 Vue 传来的剪辑参数，利用打包时内嵌的 `ffmpeg.exe` 执行物理剪辑。
* **路径解析**：获取 `app.getAppPath()` 拼接出内嵌 `ffmpeg.exe` 的绝对路径。
* **指令生成示例**（精确裁剪并重新编码，确保兼容性）：
  ```bash
  # 从源视频截取 00:00:05 到 00:00:15 的片段
  ffmpeg -i "C:\source.mp4" -ss 00:00:05 -to 00:00:15 -c:v libx264 -c:a aac "D:\Cinecho\Outputs\clip_1.mp4"
  ```
* **进度捕获**：监听 fluent-ffmpeg 的 `progress` 事件，将 `percent` 字段通过 `sync:taskProgress` IPC 通道实时发送给 Vue 的进度条组件。

## 5. 异常处理与降级体验 (Edge Cases)

* **密钥无效或欠费**：火山接口返回 401 或 403 时，主进程立即标记任务 `FAILED`，将 Error Log 写入 JSON，并向 Vue 推送前端报错弹窗（“API 鉴权失败，请检查设置或火山引擎账户余额”）。
* **磁盘空间不足**：在调用流式下载或 FFmpeg 剪辑前，主进程需通过 `node-disk-info` 等库预检输出盘符的剩余空间，低于 2GB 直接阻断并提醒。
* **应用意外退出重启恢复**：应用再次启动时，`dataManager.ts` 读取 `storage.json`。遍历 `tasks` 数组，如果发现有状态仍为 `PROCESSING` 的任务，直接静默重启该任务的 API 轮询逻辑，实现“断点续传”。
